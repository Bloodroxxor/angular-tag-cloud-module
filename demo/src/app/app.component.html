Simple Example Setting up the TagCloud with minimum propertys This exaple passes TagCludData into the [data]-property. [width] and [height] are set to a value which is interpreted as pixel size in this case.

[data] has to be an Array of CloudData (interface is exported by the module). CloudData must contain the attribute 'text' which has to be a string. It may contain the following attributes:

Property Description
weight seting the value for weight which is relative to other elements on a discret scale from 1 to 10
link setting an URL string which is the target of the link
external Open a link in a new Tab (property link has to be set)
color setting a CSS color string
Using relative width Value <= 1 for [width] causes the usage of relative width Setting the [width] to a value which is less or equal to 1 causes the usage of relative with for the TagCloud. E.g. 0.8 means a cloud width of 80%. No overflow Setting [overflow] to 'false'. Setting the [overflow] to 'false' causes that if words are just show if they fit into the container. Catch & handle events Use (clicked), (afterInit) and (afterChecked) You can catch and handle (clicked), (dataChanges), (afterInit) and (afterChecked) events which are fired by the TagCloudComponent. The (clicked) event returns the element of type CloudData that you clicked on. The (dataChanges) object returns an object of type SimpleChanges (interface provided by @angular/core). You have access to the previous and current values. Rotate some words Set the attribute 'rotate' for an CloudData element to a deg value for rotation You can set the attribute 'rotate' for an CloudData element to a deg value for rotation to rotate an element by the given deg value. Zoom words on hover Set the attribute 'zoomOnHover' of type `ZoomOnHoverOptions` zoom elements on hover. You can set the attribute '[zoomOnHover]' with an Object of type 'ZoomOnHoverOptions'.
